A few notes on using this:

**Paste it as a single message.** Don't break it into separate Replit Agent sessions or it will lose context between changes.

**Watch change 3 (Clone) most carefully.** That's the most complex — it touches the API, the route handler, the list UI, and the wizard. If Replit Agent starts suggesting schema changes to support it, stop it and tell it the clone must work by reading the existing quote fields and posting a new quote via the existing create endpoint only.

**The codegen rule is important.** If Agent touches `openapi.yaml` it will then want to run codegen, which can cascade. The clone endpoint may need a new route — if so, instruct it to add the route directly in the Express handler and note it as an exception in the API spec comment, matching the pattern already used for the feedback submission.
